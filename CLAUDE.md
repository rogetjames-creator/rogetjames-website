# ROGETjames Website — Claude Context

## Project
Premium laser-cut wall art and sculpture studio website. React SPA built with Vite, Tailwind CSS v4, GSAP animations, deployed on Netlify.

**Live site:** rogetjames.com  
**Active branch:** `claude/remote-status-check-re48a5`

## Stack
- React 19 + Vite (multi-page: main site + vault)
- Tailwind CSS v4 (`@tailwindcss/vite`)
- GSAP 3 + ScrollTrigger (animations)
- Lenis (smooth scroll, main site only)
- Netlify + Netlify Functions (serverless backend)

## What has been built

### Client Vault System
A personalised, locked page for each client — sent via email, verified by email address.

**Flow:**
1. Roger adds a client row to Airtable (`Clients` table)
2. Roger visits `/vault?admin=1`, enters admin password, types client email → hits Send
3. Client receives branded Resend email with link → `rogetjames.com/vault?token=TOKEN`
4. Client enters their email to verify identity → vault unlocks
5. Client sees their exclusive page; footer CTA draws them to the main site
6. LocalStorage caches the session so they can revisit without re-entering email

**Key files:**
- `vault.html` — separate HTML entry point for vault page
- `src/vault.jsx` — React entry for vault
- `src/components/VaultPage.jsx` — full vault UI (all components in one file)
- `netlify/functions/vault-verify.js` — POST `/api/vault-verify` — checks email+token against Airtable, returns client content
- `netlify/functions/vault-invite.js` — POST `/api/vault-invite` — admin sends invite email via Resend

**Vault page sections** (only render if content exists):
- Full-screen hero slideshow (GSAP crossfade, progress bar, slide dots)
- Sticky header (appears on scroll via ScrollTrigger)
- Project overview (greeting, description, status badge)
- Gallery slideshow (GSAP horizontal slide transitions, click for fullscreen lightbox)
- Key Points grid (scroll-triggered stagger animation)
- Links/resources cards (hover animations, external/internal)
- PDF document downloads
- Footer CTA → main site

**Admin panel** at `/vault?admin=1`:
- Password-protected (checks `VAULT_ADMIN_SECRET` env var server-side)
- Enter client email → sends invite → shows vault URL to copy

### Airtable CMS structure
Table name: `Clients`

| Field | Type | Notes |
|---|---|---|
| Name | Single line | Client/project name |
| Email | Email | Client email |
| Token | Single line | UUID — auto-generated by invite function, do not edit |
| Project Title | Single line | e.g. "Garden Sculpture Commission" |
| Project Description | Long text | Shown in overview section |
| Location | Single line | e.g. "Perth, WA" |
| Status | Single select | Design / In Progress / Review / Complete / Delivered |
| Greeting | Single line | Opening message shown as a pull quote |
| Images | Attachments | All project images — used for hero slideshow + gallery |
| Key Points | Long text | One point per line |
| Links | Long text | One per line: `Label\|URL` or `Label\|URL\|Description` |
| PDFs | Attachments | Downloadable documents |
| Invite Sent | Checkbox | Auto-set when invite is sent |

### Netlify environment variables required
```
AIRTABLE_API_KEY       — Airtable personal access token
AIRTABLE_BASE_ID       — Base ID (starts with "app…")
AIRTABLE_TABLE_NAME    — Optional, defaults to "Clients"
RESEND_API_KEY         — Resend API key
VAULT_ADMIN_SECRET     — Password for /vault?admin=1
VAULT_FROM_EMAIL       — e.g. "ROGETjames <james@rogetjames.com>"
```

## Build
```bash
npm run dev          # Vite dev server, port 5176
npm run build:quick  # Vite build only (used by Netlify)
npm run build        # Vite build + Playwright prerender
```

Vite is configured as a multi-page app — `index.html` (main site) and `vault.html` (vault) are both entry points.
